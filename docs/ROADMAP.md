## 1 · STRATEGIC ARCHITECTURE (LOCKED)

### 1.1 What House Haven Realty Is

House Haven Realty is a licensed Tennessee real estate brokerage based in Nashville, led by broker/owner Stephen Delahoussaye, with a team of ten agents and staff. It is boutique, local, and human. It is not a platform. It is not a tech startup. It is not a tools company. It is a brokerage — one that happens to be building better tools than its Nashville competitors because better tools serve its clients better.

Revenue comes from one source only: closings from inbound leads generated by the website. Every strategic decision flows downstream from that fact.

### 1.2 The Core Promise

> A real estate brokerage that adds value first, accolades last, and real estate knowledge intertwined.

Every design, copy, and feature choice should pass the test: *does this expression add value to a Nashville buyer or seller before asking for anything?* If no, cut or rework it.

### 1.3 The Three Hero Users

All site design decisions trace back to these three people. When in doubt about a UX call, ask which of them is being served.

**Chris from The Nations** — 32, renting in The Nations with his wife and a 2-year-old, second baby coming. Pre-approved for $475K. Needs education on the buying process, FTHB programs, and what new construction is hitting the market near him. Not ready today, will be in 90 days. Searches: "homes for sale in the Nations," "Nashville FTHB programs," "new construction Nashville." Values honest guidance over sales pressure.

**Sarah in West Nashville** — 41, owns in Sylvan Park, bought in 2018 for $410K, now worth ~$720K. Curious what her home is worth. Thinking about selling in 12–18 months, not committed. Searches: "what's my home worth Nashville," "Sylvan Park home values." Needs a reason to trust a specific agent over Zillow's estimate and Gary Ashton's billboard.

**The Thompsons** — late 30s, relocating from Atlanta. Both working professionals, combined income $300K, budget $850K–$1.1M. Actively shopping, looking at Franklin, Brentwood, East Nashville. Want new construction or lightly-lived-in homes. Searches: "new homes Franklin TN," "Brentwood new construction," "Nashville real estate agent." Need an agent who knows the new-build landscape cold.

### 1.4 The Architecture (Non-Negotiable)

1. **Homepage is brokerage-first.** First screen passes the 0.05-second test: is this a Nashville real estate brokerage? Yes. Hero, two CTAs (search homes, what's my home worth), trust markers in one line, live MLS listings strip.
2. **Nashville Pipeline is the flagship differentiator.** Second content block on the homepage. Card in the "House Haven difference" grid. Its own full page at `/pipeline`. Its own SEO/PR engine. It is what makes House Haven different — it is not what House Haven *is*.
3. **Tools layer in over time.** Launch with homepage + live MLS + Nashville Pipeline (already built, rebranded) + one seller tool (House Haven Value). Nothing else at launch. Every 2–3 weeks after, one additional tool ships with a shareable moment.
4. **Every tool is named "House Haven [Thing]"** and framed as "what this brokerage built for you." The brokerage is always the subject. The tool is always the gift.

### 1.5 Competitive Position

Incumbent: Gary Ashton's nashvillesmls.com — $5B lifetime sales, 170+ agents, 50+ microsites, 8,000–10,000 daily visitors, 100+ inquiries/day, "Don't Sell Without the Intel" branding everywhere. You don't out-Ashton Ashton. You can't outspend him on radio or billboards or match 20 years of domain authority in 12 months.

You can:
- Out-brand him (his is 2001-era internet, House Haven is 2026)
- Beat him on tools (his "Intel" is MLS pre-list access; Pipeline is live permit data — different product)
- Beat him on experience (boutique feels different than 170 agents)
- Beat him on content depth (deep on 65 communities vs thin across 50 microsites)

Positioning line: *Compass-restrained × Public.com-approachable × Nashville-native warm.*

### 1.6 The Brand Voice

Direct, warm, informed. Short sentences. Specific facts over vague claims. **Banned:** "luxury," "world-class," "premier," exclamation marks in headlines, stock-photo smiling-couple-with-keys, exit-intent popups, chatbots, email walls in front of tools.

If a sentence could have been written by any Nashville brokerage, rewrite it or cut it.

### 5.2 NAR IDX Policy + 2026 Settlement (UPDATED)

- Realtracs copyright on every page that displays MLS data
- "Information deemed reliable but not guaranteed" disclaimer
- Last-updated timestamp visible
- Listing attribution (listing agent/office) on every listing detail
- **No display of buyer agent compensation from MLS data** (NAR-prohibited; violation terminates feed access)
- **NEW for 2026 (canonical NAR settlement language, effective Aug 17 2024):** Conspicuous disclosure on seller-facing tools (Value page, Sellers page, listing-detail seller-info sections), and inside listing agreements where not on a government-specified form:
  > **Broker commissions are not set by law and are fully negotiable.**

  Render at minimum 14px, on contrasting background, not behind a disclosure expand-toggle. Same wording sitewide so HubSpot lead notes can quote it consistently.
- Honor seller opt-out and delayed marketing designations
- Data refresh ≥ 24 hours (we refresh every 15 minutes via ISR)

### 5.3 Fair Housing Act

- Equal Housing Opportunity logo in footer
- Community page content reviewed for steering — describe amenities, commute, schools as data; **never demographics**
- Diverse imagery in stock selections
- No "good neighborhood" or "safe area" language without data backing

### 7.3 Technical Implementation

**Routes**
- `app/value/page.tsx` (server component for SEO)
- `app/value/ValueClient.tsx` (client component for interactivity)
- `app/api/value/route.ts` — POST, body `{ address, lat, lng }`. Server-side RentCast call (never expose API key to client). Cache in Supabase `valuation_cache` with 30-day TTL. Returns `{ estimate: { low, mid, high }, comps: [...], confidence: string }`.

**RentCast client (`lib/rentcast.ts`)**

```typescript
export async function getValuation(address: string, lat: number, lng: number) {
  const response = await fetch(
    `https://api.rentcast.io/v1/avm/value?address=${encodeURIComponent(address)}`,
    { headers: { 'X-Api-Key': process.env.RENTCAST_API_KEY!, 'Accept': 'application/json' } }
  );
  if (!response.ok) throw new Error('Valuation unavailable');
  const data = await response.json();
  return {
    mid: data.price,
    low: data.priceRangeLow,
    high: data.priceRangeHigh,
    comps: data.comparables?.slice(0, 5) ?? []
  };
}
```

**Supabase tables**

```sql
CREATE TABLE valuation_cache (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  address_normalized text UNIQUE NOT NULL,
  latitude double precision,
  longitude double precision,
  estimate_low numeric,
  estimate_mid numeric,
  estimate_high numeric,
  comps jsonb,
  fetched_at timestamptz DEFAULT now()
);
CREATE INDEX idx_valuation_cache_address ON valuation_cache(address_normalized);
CREATE INDEX idx_valuation_cache_fetched ON valuation_cache(fetched_at);

CREATE TABLE cma_requests (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name text NOT NULL,
  email text NOT NULL,
  phone text,
  address text,
  estimated_value_mid numeric,
  timeline text,
  tcpa_consent boolean DEFAULT false,
  created_at timestamptz DEFAULT now(),
  hubspot_contact_id text,
  notified_at timestamptz
);
```
